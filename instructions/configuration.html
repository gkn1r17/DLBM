Basic Configuration

DLBM loads settings through config.Config, which combines scientific settings, control settings, and optional artificial transport-matrix settings. Settings can come from default files, a supplied settings file, or command-line overrides.

In practice, the main groups you are likely to edit are:

Time stepping: DURATION_DAY, GROWTH_HOURS, DISP_HOURS
Population and ecology: K, INIT_LIN_SIZE, GROWTH_RATE_DAY, MORTALITY_DAY, MUTATION
Input/output: FILE_OUT, FILE_LOAD, SAVE_TIMESTEPS_DAY, REPORT_TIMESTEPS_DAY, MUTANT_TIMESTEPS_DAY
Parallelization: CLUST_FILE and NUMNODES

Typical Run Pattern

NUMNODES 1
SETTINGS testSettings/testNeutral.ini
DURATION_DAY 365
FILE_OUT testResults/testNeutral
SEED 100

Command-line values take precedence over values from the settings file. Unknown settings are rejected, which helps catch spelling mistakes early.

Adding a New Transport Matrix

A new transport matrix usually requires three coordinated inputs:

Connectivity data via TM_FILE
Relative box volumes via VOL_FILE
Environmental forcing via TEMP_FILE when selection is enabled

The configured number of boxes should match the transport data unless you are using the artificial matrix builder. If you want a simple idealised system instead, the artificial TM pathway under transportMatrix.artificialTM can be used to generate locations dynamically.

After adding a new matrix, it is worth running a short simulation first to confirm that loading, box counts, and output filenames all look correct before moving to a longer cluster run.
