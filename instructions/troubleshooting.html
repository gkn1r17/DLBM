Common Error Messages

If the program reports unknown settings, start by checking spelling and section placement in the settings file. Configuration parsing is deliberately strict and will fail on leftover or misspelled keys.

If a load file cannot be found, verify both the base filename and whether the run expects a dated output suffix. Restart logic can infer the latest matching run, but only when filenames follow the expected pattern.

Message Parsing (MPI) Errors

MPI-related failures are often caused by mismatched assumptions between local and distributed code paths, or by changes that affect the structure of communicated data without updating both send and receive logic. When debugging, reduce the problem first to a minimal run with a small number of nodes and simple settings.

Memory Errors

Memory pressure usually grows with box count, lineage count, mutation activity, and the amount of saved state. Useful first responses include shortening the run, reducing save frequency, disabling optional outputs, or simplifying the test case.

When adding new lineage properties, treat memory use as part of the feature design rather than only a runtime concern.
