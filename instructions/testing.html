General Testing Procedure

The existing JUnit coverage runs scenario-based regression tests from src/test/TestJUnit.java. These tests execute representative runs, compare new results with baseline output, then check whether saved output can be reloaded correctly.

A sensible workflow after a code change is:

Run a short local scenario first
Confirm output files are created as expected
Run the relevant JUnit regression tests
Only update baselines when the behavioural change is intentional

Adding Additional Tests

New tests are easiest to maintain when they focus on one scenario or one feature branch at a time. If you add new output formats or diversity dimensions, include both a direct behaviour test and a save/load test whenever possible.

Keep seeds fixed in regression tests so failures are easier to interpret, and use separate settings files when a feature needs a distinct configuration surface.
