System Requirements (already present on Iridis 6 cluster)

JavaSE-1.8 or higher
FastMPJ if running on cluster

Quick Start Steps

assuming Linux SLURM cluster (e.g. Iridis at Southampton) or local Linux environment, may need to be modified for other OS or other cluster environments.

1. Move the "run" folder into the directory from which you want to run. All commands should be executed from within this folder.
2. Navigate to the run folder on the server.
3. If running on cluster: Modify runLBM.slurm as required:
- replace "gkn1r17" with your username.
- modify "nodes" and "cpus-per-task" according to number of nodes and cpus available on your cluster
- modifying number of nodes will also require a new cluster file, see parallelization section
- modify "time" according to time available on server
- modify "mem" according to RAM (kb) available on server
4. Choose the appropriate run command (see below) for either distributed SLURM execution or a local Linux run.
5. Monitor run progress in log files ("slurm-*.out" files on SLURM cluster ).
6. If server timeout or run failure occurs before run has finished see "Running from checkpoint pickup" (below).
7. Once run has finished collate results using "Collating" (below) commands.

To run a simulation with different parameters, either: 1. create a new settings .ini file.
- use default.ini, testSettings/testSelective.ini and testsettings/testNeutral.ini as a guide.
- change SETTINGS [your_settings_file] in run command, whether running locally or on cluster.
2. run with additional settings as command line arguments (these take precedence over ini file settings).

Running on SLURM Cluster

- Adjust NUMNODES and CLUST_FILE if environment has a different number of nodes. 

sbatch runLBM.slurm LBM.jar NUMNODES 2 SETTINGS testSettings/testMutation FILE_OUT phylo3/mut CLUST_FILE clusters6386.csv

Running Locally On Linux

Before the first run, make the script executable: 

chmod +x instructions/runLBM.local.sh

Then call it as follows (note NUMNODES 1 and CLUST_FILE clusters6386nondist.csv as this is a non distributed i.e. one node environment): 

bash ./runLBM.local.sh LBM.jar NUMNODES 1 SETTINGS testSettings/testMutation CLUST_FILE clusters6386nondist.csv FILE_OUT phylo3/mut

Running from checkpoint pickup

Add FILE_LOAD to the command line or ini file. This example is for SLURM but same principle for running locally.
Automatically the last complete pickup file, these are output csv files ending in CHKA or CHKB will be loaded.
You can see what day this corresponds to at the top of these files.
To load a specific day / hour add e.g. "LOAD_HOUR 365" or "LOAD_HOUR 365h8" (if loading a timestep part way through the day, in this example 8 hours in)

sbatch runLBM.slurm LBM.jar NUMNODES 2 SETTINGS testSettings/testMutation FILE_OUT phylo3/mut CLUST_FILE clusters6386.csv FILE_LOAD phylo3/mut

Collating

1. Modify runConvert1.sh if required : the numbers in the for loop are timestamps (in days) to include in mat file, by default equal to timesteps.csv. (If extra days are in this loop it won't cause an error but make sure all days you need are at least included)
2. Run runConvert1.sh using the command using the below example below as a guide. This converts the output for each timestamp from csv to mat. The format of this command are:
sh ./runConvert1.sh [path to csvs] [desired output path/name] [name in csvs, including date] [hour-min-second part of name of csvs]
3. When this has finished for every timestamp, run runConvert2.sh
sh ./runConvert2.slurm [path to output] [output name]
(note these scripts are written for SLURM cluster and will need adjusting if running locally) 

sh ./runConvert1.sh testSelective results2026/W=6dispK1000M0 seasW6_T15-04-2026 21-37-43
sbatch runConvert2.slurm results2026 W=6dispK1000M0
