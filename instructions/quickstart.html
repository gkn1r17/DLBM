Quick Start

First steps for launching a DLBM run

System Requirements

The current run workflow is aimed at a Linux-based distributed cluster using SLURM for job submission. The original quick start was written around the Iridis cluster at the University of Southampton.

You will need:

JavaSE-1.8 or higher
Linux for both cluster and local shell-based runs
SLURM and MPI for distributed runs

A single-computer run is also possible if you use a one-machine cluster file and set the node count to 1.

Quick Start Steps

1. Move the run folder onto the target server or cluster.
2. Create a settings file using testSettings/testNeutral.ini for a neutral simulation or testSettings/testSelective.ini for a selective simulation as a guide.
3. Navigate to the run folder on the server.
4. Choose the appropriate run command for either distributed SLURM execution or a local Linux run.

Running On A SLURM Cluster

Use the distributed launcher when running on a Linux cluster with SLURM and MPI available. Adjust the jar name, output prefix, settings file, and node count to match your run. 

sbatch runLBM.slurm LBMredone3.jar NUMNODES 2 SETTINGS testSettings/testMutation FILE_OUT phylo3/mut

sbatch runLBM.slurm LBMredone3.jar NUMNODES 2 SETTINGS testSettings/testMutation FILE_OUT phylo3/mut FILE_LOAD phylo2/mut

sbatch runLBM.slurm LBMredone3.jar NUMNODES 2 SETTINGS testSettings/testMutation FILE_OUT phylo3/mut FILE_LOAD phylo2/mut LOAD_DAY 365000

The NUMNODES value should match the number of top-level clusters or computers you intend to use. Use command-line overrides to add or replace individual settings as needed.

Running Locally On Linux

For a local Linux run, use runLBM.local.sh, which is based on the SLURM launcher but runs the jar directly with java. This script forces the local settings override NUMNODES 1 and CLUST_FILE clusters6386nodist.csv.

Before the first run, make the script executable: 

chmod +x instructions/runLBM.local.sh

Then call it with the jar path first, followed by the normal key-value run arguments: 

bash instructions/runLBM.local.sh LBMredone3.jar SETTINGS testSettings/testMutation FILE_OUT phylo3/mut

bash instructions/runLBM.local.sh LBMredone3.jar SETTINGS testSettings/testMutation FILE_OUT phylo3/mut FILE_LOAD phylo2/mut

bash instructions/runLBM.local.sh LBMredone3.jar SETTINGS testSettings/testMutation FILE_OUT phylo3/mut FILE_LOAD phylo2/mut LOAD_DAY 365000

You do not need to pass NUMNODES or CLUST_FILE for the local script because it appends the local values automatically. If you need different Java memory settings, set JAVA_OPTS before running the script.

Expected Output

After submission, the run produces output in the configured output folder together with run metadata.

Lineage abundance output: CSV files for each regular reporting interval
Settings snapshot: a copy of the settings used for the run
Seed marker files: empty files whose names record the random seeds used for each cluster
SLURM log output: a slurm-[job id].out file in the run folder

By default, abundance summaries are written at regular intervals defined in the settings file.

Run-Time Notes

Change SETTINGS testSettings/testMutation as appropriate for the run you want to perform. See default.ini for examples of the available settings, and add one-off overrides on the command line in pairs such as DURATION_DAY 3650000.
