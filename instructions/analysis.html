Model Output

Full results

Full population saves are written at configured save times. These files contain the detailed state needed for restart, comparison testing, or deeper external analysis.

Time series

The reporting pathway writes local diversity by box and global diversity summaries over time. These outputs are often the quickest way to check whether a run behaves sensibly before loading larger files.

Phylogenies

When mutation output is enabled, mutant history is saved separately. This is useful for lineage ancestry analysis, but it should be treated as an optional output stream rather than something guaranteed for every run.

Running the Output Collation Script

The original outline refers to a downstream collation step for Matlab-style analysis. In this draft, the recommended workflow is to gather time-series CSVs, full save files, and any mutant files into a dedicated run folder before collation so the data products for one scenario remain together.

Structure of Collated Output

Expect collated data to separate:

Run metadata such as seed, timestamps, and settings snapshot
Global summaries such as total diversity over time
Local summaries by grid box
Optional phylogeny-derived products when mutation tracking is active

If you extend the simulation output, it is helpful to keep derived analysis tables versioned so older post-processing scripts remain interpretable.
