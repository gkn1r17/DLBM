Additional Dimensions of Diversity

New types of Lineage (associated with new information e.g. size) can be added as new classes in the lineages package extending Lineage (see SelLineage) as an example.
For initialisation change transportMatrix.GridBox.initIDSizeTemp(...) and transportMatrix.GridBox.initPop(...)
Change [My-new-lineage].getDetails(...) to output new information. This will add column to output CSV files.
Add to Lineage.makeNew(...)
Adjust increment in GridBox.addLoadedPop(...) to correctly load new information.
If running distributed modify decompileMPImessage compileMPImessage (and if testing locally these methods + "test") so that new information is correctly tranferred between nodes and Lineage.makeNewFromDist(...)
For storing the new information it can either be done in the new Lineage class and will need to be copied with every dispersal or follow the example of temperature with RunState.tempLins to save memory.

Adding new Configuration settings

Scientific settings should be added to config.SciConfig
Control settings should be added to config.ControlConfig
Settings dependent on both scientific and control settings should be added directly to config.Config
